# Specification: 年度打卡热力图看板

## 1. 概述
在 `/summary` 统计页面中增加一个年度热力图（Heatmap），类似 GitHub 贡献图。该图表将直观展示用户过去一年的每日打卡情况，颜色越深代表当天的俯卧撑数量越多。

## 2. 核心功能
*   **热力图展示**：
    *   展示过去 365 天的打卡数据。
    *   以周为列，以星期为行（周一至周日）。
    *   每个格子代表一天。
*   **颜色分级**：
    *   **Level 0 (无数据)**: 灰色背景。
    *   **Level 1 (少)**: 浅绿色。
    *   **Level 2 (中)**: 中绿色。
    *   **Level 3 (多)**: 深绿色。
    *   **Level 4 (极多)**: 极深绿色。
    *   *注：具体阈值可动态计算或固定设置（例如：1-30, 31-60, 61-100, >100）。*
*   **交互**：
    *   **Tooltip**：鼠标悬停（或点击）在格子上时，显示具体日期和俯卧撑数量（例如：“2026-01-16: 50 个”）。
*   **移动端适配**：
    *   由于手机屏幕宽度有限，无法完整展示 52 周。
    *   **方案**：在移动端仅展示最近 3-4 个月，或者支持水平滚动查看完整热力图。

## 3. 技术实现
*   **组件**：新建 `components/heatmap.tsx` 客户端组件。
*   **数据获取**：复用或扩展 `app/summary/actions.ts`，增加获取年度数据的方法 `getYearlyPushups`。
*   **样式**：使用 Tailwind CSS Grid 布局实现网格。

## 4. 验收标准
1.  PC 端 `/summary` 页面正确显示完整年度热力图。
2.  移动端 `/summary` 页面显示适配后的热力图（横向滚动或缩减范围）。
3.  不同数量的打卡记录显示不同深浅的颜色。
4.  悬停/点击格子能准确显示日期和数量。
